Computers can't code

No matter how much AI you spread on 'em

Posted on Dec 9, 2022 3 mins read

Following are a few principles to guide discussion about programming and artificial intelligence.

  1. Code is not the language of computers. It is the language of extremely specific humans.
  2. A perfect AI would refuse to write your code for you.
  3. Any programmer who thinks their job could be done by an AI is working too hard.

I will explain.

Code is not for computers

Much of the excitement about AI-generated code comes from the promise of accessibility. Surely if an AI can handle the magic words and brackets, non-technical folk will no longer be held back from creating apps, right? (And perhaps we can fire those six-figure lackeys in the software department and save some payroll.)

But keywords and curly braces aren’t what’s hard about programming. Programming languages do not have a vast symbology. Typically they comprise 30 to 50 words and a handful of special characters. Add in a couple hundred common API methods on any given platform and you still don’t approach the level of human language. If you knew as much Python as high school Spanish, you’d know enough.

Code isn’t demanding because of arcane words and phrases. It’s demanding because of the level of detail you have to communicate in. And those details can’t be delegated.

You may think you want something simple: a text field that is saved to, and retrieved from, a database. You can pay a programmer to build this. They’ll likely fire back with a few questions—I can think of several. What should the field look like? What languages does it need to support? Should it save automatically every second or so, or only when the user clicks a button? How much text should the user be allowed to input? Will the text contain sensitive information? Will anyone else need to retrieve it? How should we identify the user so we can give their text back to them later?

In the context of long-term employment, the programmer can ask significantly fewer questions. They draw on context. This doesn’t mean they’re making fewer decisions. No, all the decisions still have to be made.

Those decisions are laid out in lines of code. And in modern programming languages (anything from the last few decades, really) they resemble the registers and instruction sets of a computer far less than they resemble a well-written recipe or essay. Almost the entire fact of the computer has been abstracted away from us. We describe where data should come from and what should be done with it—in incredible detail, of course—and let the compiler do the rest.

Computers are predictable devices, like hammers. They’ll “speak” whatever language we teach them.

A programmer’s job isn’t to translate from English to beep-boop. It’s to build a forest, one pine needle at a time.

An AI is not a prophet
